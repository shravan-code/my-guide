Data Formats

CSV, JSON, Parquet, Avro, and more — understanding data serialization formats.

CSV / TSV Basic

Comma-Separated Values (CSV) and Tab-Separated Values (TSV) are simple text formats for tabular data.

Pros & Cons

  • Pros: Human-readable, universal support, simple
  • Cons: No schema, large files, slow parsing
  • Use case: Data exchange, quick exports

Example

id,name,email,age 1,Alice,alice@example.com,28 2,Bob,bob@example.com,35 3,"Last, First",charlie@example.com,42 # TSV (Tab-separated) id name email age 1 Alice alice@example.com 28
# Python CSV reading import csv with open('data.csv') as f: reader = csv.DictReader(f) for row in reader: print(row['name']) # Pandas CSV reading import pandas as pd df = pd.read_csv('data.csv')

JSON Basic

JavaScript Object Notation (JSON) is a lightweight format for semi-structured data with nested structures.

Pros & Cons

  • Pros: Flexible schema, nested data, web-native
  • Cons: Verbose, no data types, slow for analytics
  • Use case: APIs, config files, document stores

Example

{ "user": { "id": 123, "name": "Alice", "roles": ["admin", "editor"], "address": { "city": "Seattle" } } }
# PostgreSQL JSONB storage CREATE TABLE events ( id SERIAL PRIMARY KEY, payload JSONB NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW() ); # Query JSON fields SELECT payload ->> 'user' AS user_name FROM events WHERE payload @> '{"user":{"roles":["admin"]}}'::JSONB;

XML Basic

Extensible Markup Language (XML) is a markup format for hierarchical data with schemas.

Pros & Cons

  • Pros: Self-describing, schema validation, namespaces
  • Cons: Verbose, complex parsing, slow
  • Use case: Enterprise integration, SOAP APIs

Example

<?xml version="1.0" encoding="UTF-8"?> <user id="123"> <name>Alice</name> <email>alice@example.com</email> <roles> <role>admin</role> <role>editor</role> </roles> </user>

YAML Basic

YAML Ain't Markup Language - a human-friendly data serialization format.

Pros & Cons

  • Pros: Very readable, supports comments, anchors
  • Cons: Indentation-sensitive, slow parsing
  • Use case: Config files, CI/CD pipelines, Kubernetes

Example

# User configuration user: id: 123 name: Alice email: alice@example.com roles: - admin - editor settings: theme: dark notifications: true

Parquet Intermediate

Apache Parquet is a columnar storage format optimized for analytics workloads.

Pros & Cons

  • Pros: Excellent compression, column pruning, predicate pushdown
  • Cons: Not human-readable, slower writes
  • Use case: Data lakes, analytics, Spark/Hadoop

Columnar vs Row

# Row storage (CSV): Scan ALL columns [id, name, email, age] [id, name, email, age] [id, name, email, age] # Columnar (Parquet): Scan ONE column [id, id, id] # Only read id [name, name, name]
# Python Parquet import pandas as pd # Write df.to_parquet('data.parquet') # Read specific columns (column pruning) df = pd.read_parquet('data.parquet', columns=['id', 'name']) # PySpark df = spark.read.parquet('s3://bucket/data.parquet')

ORC Intermediate

Optimized Row Columnar (ORC) is a columnar format optimized for Hive and Oracle ecosystems.

Pros & Cons

  • Pros: High compression, ACID support, indexes
  • Cons: Hive-centric, less tool support than Parquet
  • Use case: Hive data warehouses, Oracle integration

ORC Structure

# ORC file structure ┌─────────────────────┐ │ File Footer │ │ - Column statistics │ │ - Stripe statistics │ ├─────────────────────┤ │ Stripe 1 │ │ - Row data (column) │ │ - Indexes │ ├─────────────────────┤ │ Stripe 2 │ │ ... │ └─────────────────────┘

Apache Arrow Intermediate

Arrow is an in-memory columnar format designed for zero-copy reads and interoperability.

Pros & Cons

  • Pros: Zero-copy between languages, fast analytics
  • Cons: In-memory only, large RAM footprint
  • Use case: DataFrame interchange, OLAP engines

Use Cases

  • Polars (Rust DataFrame library)
  • Apache Spark (Arrow-based UDFs)
  • DuckDB (in-process analytics)
  • Pandas 2.0 (Arrow backend)
# Python Arrow import pyarrow as pa import pandas as pd # Convert Pandas to Arrow (zero-copy) table = pa.Table.from_pandas(df) # Convert Arrow to Pandas (zero-copy) df_back = table.to_pandas() # Write Arrow IPC format with pa.OSFile('data.arrow', 'wb') as f: with pa.IPCWriter(f) as writer: writer.write_table(table)

Avro Intermediate

Apache Avro is a row-based binary format with embedded schema, popular for Kafka and data pipelines.

Pros & Cons

  • Pros: Schema evolution, compact, fast writes
  • Cons: Not columnar, slower for analytics
  • Use case: Kafka, streaming, message queues

Schema Example

{ "type": "record", "name": "User", "fields": [ {"name": "id", "type": "int"}, {"name": "name", "type": "string"}, {"name": "email", "type": ["null", "string"]} ] }

Protocol Buffers Advanced

Protocol Buffers (protobuf) is Google's language-neutral, platform-neutral serialization format.

Pros & Cons

  • Pros: Very compact, fast, strongly typed
  • Cons: Binary (not human-readable), schema required
  • Use case: gRPC, microservices, performance-critical

Proto Definition

syntax = "proto3"; message User { int32 id = 1; string name = 2; string email = 3; repeated string roles = 4; }

SQL Dump Basic

SQL dumps export database schema and data as SQL statements for backup and migration.

# PostgreSQL dump pg_dump -h localhost -U myuser mydb > backup.sql # Compressed dump pg_dump mydb | gzip > backup.sql.gz # Custom format (compressed, flexible restore) pg_dump -Fc mydb > backup.dump # Restore psql mydb < backup.sql pg_restore -d mydb backup.dump # Dump structure only pg_dump -s mydb > schema.sql # Dump data only pg_dump -a mydb > data.sql

Backup & Restore Flow

pg_dump → SQL File → Version Control → psql / pg_restore

Delta Lake Advanced

Delta Lake is an open-source storage layer that brings ACID transactions to data lakes.

Features

  • ACID transactions: Consistent reads/writes
  • Time travel: Query previous versions
  • Schema enforcement: Prevents bad data
  • Upserts: MERGE INTO support

Usage

# PySpark with Delta from delta import configure_spark_with_delta_pip df.write.format("delta") \ .mode("overwrite") \ .save("/delta/users") # Time travel df_old = spark.read.format("delta") \ .option("versionAsOf", 5) \ .load("/delta/users")

Apache Iceberg Advanced

Iceberg is a high-performance table format for huge analytic datasets with hidden partitioning.

Features

  • Hidden partitioning: No manual partition columns
  • Schema evolution: Add/rename columns safely
  • Partition evolution: Change partition strategy
  • Time travel: Snapshot isolation

vs Delta Lake

# Iceberg: Engine-agnostic # - Spark, Trino, Flink, Hive # - Open metadata format # - Hidden partitioning # Delta: Spark-centric # - Native Spark integration # - Delta Lake protocol # - Storage-level transactions
