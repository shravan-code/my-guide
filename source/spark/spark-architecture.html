1

Driver Program

The driver is the entry point that runs the main() function and creates SparkContext. It coordinates the entire job execution.

SparkContext

Connection to the cluster, manages resources and schedules tasks.

DAGScheduler

Creates DAG of stages from RDD operations.

2

Cluster Manager

Allocates resources across the cluster. Spark supports multiple cluster managers.

Supported Managers

Standalone | YARN | Kubernetes | Mesos
3

Executors

Worker processes that run tasks and store data in memory or disk. Each application gets its own executors.

Task Execution

Run tasks assigned by the driver, report results back.

Memory

Cache user data in memory for fast access.

4

Catalyst Optimizer

Tree-based query optimizer that transforms logical plans to optimized physical plans.

Optimization Flow

Unresolved Logical Plan -> Analyzed Plan -> Optimized Plan -> Physical Plans
5

Tungsten

Whole-stage code generation engine that produces optimized bytecode for query execution.

Memory Management

Sun JDK Memory Manager with off-heap storage.

Cache-aware

Optimized data structures leveraging CPU cache.

6

DAG Scheduler

Breaks the logical execution plan into stages with boundary at shuffle boundaries.

DAG Creation

RDD Operations -> Build DAG -> Create Stages -> Submit Tasks
7

Resilient Distributed Datasets (RDD)

Core immutable distributed data abstraction. Tracks lineage for fault tolerance.

Transformations

map, filter, flatMap, groupByKey, reduceByKey

Actions

collect, count, reduce, take, saveAsTextFile

8

DataFrames

Distributed collection of data organized into named columns, similar to a table in a relational database.

DataFrames provide higher-level abstraction than RDDs with Catalyst optimization benefits.

9

Datasets

Type-safe extension of DataFrames with JVM objects. Combines benefits of RDDs with DataFrame optimizations.

Available in Scala and Java. Provides compile-time type safety.

10

Memory Management

Unified memory management separates execution and storage memory with dynamic sharing.

Execution Memory

Used for shuffles, joins, aggregations, and sorting.

Storage Memory

Used for caching and internal metadata.

11

Shuffle Operations

Redistributes data across partitions, causing data to move between executors across the network.

Expensive operation - minimize shuffles and use narrow transformations when possible.
