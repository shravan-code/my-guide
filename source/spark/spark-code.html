13) Filtering Data

Filtering removes rows that don't match a condition. Spark pushes filters down into the scan operators (filter pushdown) so only relevant partitions are read — this is one of the most effective performance optimizations. Use filter() or where() interchangeably; both accept column expressions, SQL-style strings, or combined conditions.

Basic Comparison Filters

Input Table

order_id region amount status
1 APAC 120.0 delivered
2 US 95.0 pending
3 EMEA 250.0 delivered

Output Table

order_id region amount status
1 APAC 120.0 delivered
3 EMEA 250.0 delivered 
from pyspark.sql.functions import col

# filter() and where() are identical
filtered = df.filter(col("amount") > 100)
filtered = df.where("amount > 100")

# Multiple chained filters (AND)
filtered = (df
    .filter(col("amount") > 100)
    .filter(col("status") == "delivered"))

# OR with | operator
filtered = df.filter((col("amount") > 200) | (col("region") == "APAC"))

# AND with & operator
filtered = df.filter((col("amount") > 100) & (col("status").isin("delivered", "shipped")))

filtered.show()

15) Joining Data

Joins combine rows from two DataFrames based on a key column. Spark uses shuffle-based joins by default for large tables, which redistributes data across partitions — an expensive operation. For small dimension tables, wrap them with broadcast() to enable a map-side join that avoids shuffle entirely. Choosing the right join type and join order is critical for both correctness and performance.

Inner Join — Matching Keys Only

Input Table A (orders)

order_id customer_id
101 C1
102 C2

Input Table B (customers)

customer_id name
C1 Alice
C3 Bob 
# Inner join — only rows where key exists in BOTH tables
joined = orders.join(customers, "customer_id", "inner")
# Result: order 101 + Alice | order 102 → dropped (no C2 match)

joined.show()
# +---------+----------+----+
# |order_id |customer_id|name|
# +---------+----------+----+
# |      101|         C1|Alice|
# +---------+----------+----+

PostgreSQL

SELECT o.order_id, o.customer_id, c.name
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id;

17) Handling Nulls

Null represents missing or unknown data in Spark. Operations on null propagate null (e.g., 1 + null = null), and aggregate functions ignore nulls by default (sum([1, null, 2]) = 3). Always define an explicit null strategy: drop rows, fill with a value, replace conditionally, or use null-safe operators to avoid silent data-quality bugs.

Detect Nulls — isNull / isNotNull

Input Table

order_id region amount
1 APAC 120.0
2 null 95.0
3 EMEA null

Output Table

order_id region amount
2 null 95.0
3 EMEA null 
from pyspark.sql.functions import col

# Keep rows where AT LEAST ONE column is null
has_nulls = df.filter(col("region").isNull() | col("amount").isNull())

# Keep rows where ALL columns are non-null
complete = df.filter(col("region").isNotNull() & col("amount").isNotNull())

has_nulls.show()

Drop Rows with Nulls — na.drop / drop

Input Table

order_id region amount priority
1 APAC 120.0 H
2 null 95.0 M
3 EMEA null L
4 US 80.0 H

Output Table

order_id region amount priority
1 APAC 120.0 H
4 US 80.0 H 
# Drop rows where ANY column is null
no_nulls = df.na.drop()

# Drop rows where ALL columns are null (keep rows with at least one value)
df.na.drop(how="any")  # default — same as above

# Drop rows where ALL columns are null
df.na.drop(how="all")

# Drop only if specific columns have null
df.na.drop(subset=["region", "amount"])

no_nulls.show()

# ROWS: includes 2 rows before regardless of value
