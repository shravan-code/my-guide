API Concepts

REST, GraphQL, gRPC - core API concepts and patterns.

What is an API? Basic

An API (Application Programming Interface) is a set of rules and protocols that allows one software application to communicate with another. It defines the methods and data formats that applications can use to request and exchange information.

Think of an API like a waiter in a restaurant:

You (Client) → Waiter (API) → Kitchen (Server) Place order Carries request Prepares food ← ← ← Receive food Delivers response Returns result
  • You don't need to know how the kitchen works
  • The waiter knows exactly how to communicate your order
  • You get the result without seeing the internal process

API Types Basic

REST API

REST is the most widely used API architecture. It uses standard HTTP methods and is stateless — each request contains all the information needed to process it.

Core Principles:

  • Stateless — No session stored on the server; every request is independent
  • Client-Server — Clear separation between UI and data storage
  • Cacheable — Responses can be cached to improve performance
  • Uniform Interface — Consistent, predictable URL structure
  • Layered System — Can include load balancers, proxies, gateways

Example REST Request:

GET https://api.example.com/users/42 Authorization: Bearer eyJhbGc... Content-Type: application/json

Example REST Response:

{ "id": 42, "name": "Alice Johnson", "email": "alice@example.com", "role": "admin" }

GraphQL API

GraphQL is a query language for APIs developed by Facebook (Meta) in 2012. Instead of multiple endpoints, it uses a single endpoint where the client specifies exactly what data it needs.

Example GraphQL Query:

query { user(id: 42) { name email posts { title createdAt } } }

Key Insight: With REST, fetching a user's posts might require 2–3 separate requests. With GraphQL, it's a single query.

SOAP API

SOAP is a protocol (not just an architectural style) that uses XML for message formatting and typically runs over HTTP or SMTP. It is strict, formal, and has built-in error handling and security.

account_balance Banking

Financial systems and inter-bank transfers requiring high security.

medical_services Healthcare

HL7 integrations and legacy healthcare data exchange standards.

business Enterprise

Government systems and legacy enterprise service buses.

gRPC API

gRPC is a high-performance RPC framework developed by Google. It uses Protocol Buffers (protobuf) instead of JSON/XML for serialization, making it significantly faster.

hub Microservices

Internal communication between backend services in a cluster.

speed Low Latency

High-throughput systems requiring minimal overhead.

stream Streaming

Efficiently streaming large amounts of structured binary data.

WebSocket API

WebSocket provides a persistent, full-duplex communication channel between a client and server. Unlike HTTP, WebSocket allows the server to push data to the client at any time.

chat Real-time Chat

Slack, Discord, and collaborative tools requiring instant delivery.

monitoring Dashboard

Live metrics, stock tickers, and real-time performance tracking.

sports_esports Gaming

Multiplayer online games requiring low-latency state sync.

Webhooks

Webhooks are reverse APIs — instead of the client polling the server for updates, the server sends data to the client automatically when an event occurs.

payments Payments

Confirming transactions from platforms like Stripe or PayPal.

rocket_launch CI/CD

Triggering builds and deployments from GitHub Actions.

campaign Alerts

Notifying external systems of new support tickets or form entries.

Architectures Intermediate

REST API

check_circle Pros

Simplicity

Widely understood architectural style with standard HTTP semantics.

Caching

Native support for HTTP level caching for high performance.

cancel Cons

Data Payloads

Commonly suffers from over-fetching or under-fetching issues.

Real-time

No native support for push notifications or real-time updates.

GraphQL API

check_circle Pros

Query Precision

Clients fetch exactly what they need, minimizing network traffic.

Single Endpoint

Simplifies client architecture by exposing all data through one URL.

cancel Cons

Implementation

Significantly more complex to implement and maintain server-side.

Caching Complexity

Standard HTTP caching doesn't work for flexible POST-based queries.

gRPC API

check_circle Pros

Raw Speed

Uses binary Protocol Buffers and HTTP/2 for ultra-fast throughput.

Streaming

Native support for bidirectional and server-side streaming.

cancel Cons

Accessibility

Not human-readable and has very limited browser-side support.

Tooling

Requires specific Protobuf tooling and client-side code generation.

Value Proposition Basic

hub

Interoperability

Problem: Different systems built in different languages cannot naturally communicate.

Solution: APIs create a universal, platform-agnostic language between services.

account_tree

Modularity

Problem: Monolithic applications become impossible to maintain as they grow.

Solution: Build independent microservices that communicate through clean interfaces.

storage

Reusability

Problem: Rebuilding common functionality (email, payments) from scratch wastes time.

Solution: Integrate battle-tested third-party services like Stripe or SendGrid via API.

security

Security

Problem: Exposing internal databases and systems directly is a massive security risk.

Solution: APIs act as a controlled gateway with authentication and rate limiting.
