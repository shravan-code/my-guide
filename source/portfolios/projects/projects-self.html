Distributed Healthcare Data Processing Pipeline

Big Data Engineering Project

Personal Project AWS EMR & PySpark

medical_servicesHigh-Volume Patient Data Orchestration

PySpark AWS EMR AWS S3 Parquet AWS Athena AWS QuickSight

track_changesObjective

Building a distributed batch processing pipeline to handle large-scale healthcare datasets (~10GB CSV) and enable serverless analytics with high performance.

buildCore Implementation

dnsDeployed a distributed Spark cluster on AWS EMR to process multi-gigabyte healthcare datasets efficiently.

storageConverted raw unstructured CSV data into partitioned and columnar Parquet format (partitioned by hospital_id and date).

transformDesigned complex Spark transformations for patient classification and hospital-level aggregations using lazy evaluation.

queryIntegrated AWS Athena for serverless SQL querying, utilizing partition pruning to minimize data scan costs.

trending_upResults & Impact

savingsImproved storage efficiency and query performance by ~50% through columnar storage (Parquet).

speedReduced query cost and latency by implementing effective partition pruning strategies.

dashboardBuilt interactive dashboards in AWS QuickSight for real-time patient trends and hospital analytics.
