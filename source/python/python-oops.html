What is Object-Oriented Programming (OOP)?

OOP is a programming paradigm that organizes code around objects rather than functions. Objects bundle data (attributes) and behavior (methods) together, making code more organized, reusable, and easier to maintain.

Classes and Objects

A class is a blueprint for creating objects. An object is an instance of a class with actual values.

# Defining a class class Dog: # Class attribute (shared by all instances) species = "Canis familiaris" # Constructor (initializer) def __init__(self, name, age): self.name = name # Instance attribute self.age = age # Instance attribute # Instance method def bark(self): return f"{self.name} says Woof!" def __str__(self): return f"{self.name} is {self.age} years old" # Creating objects (instances) buddy = Dog("Buddy", 3) max_dog = Dog("Max", 5) print(buddy.bark()) # Buddy says Woof! print(buddy) # Buddy is 3 years old print(buddy.species) # Canis familiaris
Basic

Instance vs Class vs Static Methods

Instance Methods

Access and modify object state. First parameter is self.

Class Methods

Access class-level data. First parameter is cls. Use @classmethod.

Static Methods

No access to class/object. Use @staticmethod. Just utility functions.

class MathOperations: pi = 3.14159 # Class attribute def __init__(self, value): self.value = value # Instance attribute # Instance method def double(self): return self.value * 2 # Class method @classmethod def circle_area(cls, radius): return cls.pi * radius ** 2 # Static method @staticmethod def add(a, b): return a + b obj = MathOperations(10) print(obj.double()) # 20 print(MathOperations.circle_area(5)) # 78.54 print(MathOperations.add(3, 4)) # 7

Encapsulation

Encapsulation restricts direct access to certain attributes, protecting data integrity. Python uses naming conventions: _protected and __private.

class BankAccount: def __init__(self, balance): self.__balance = balance # Private attribute (name mangling) self._account_id = "ACC123" # Protected attribute # Getter def get_balance(self): return self.__balance # Setter with validation def deposit(self, amount): if amount > 0: self.__balance += amount return True return False # Property decorator for clean access @property def balance(self): return self.__balance account = BankAccount(1000) print(account.get_balance()) # 1000 account.deposit(500) print(account.balance) # 1500 (using property) print(account._account_id) # ACC123 (accessible but discouraged) # print(account.__balance) # AttributeError!

Tip: Use @property to create managed attributes with getters/setters.

Inheritance

Inheritance allows a class (child) to inherit attributes and methods from another class (parent), promoting code reuse and creating hierarchical relationships.

# Parent class class Animal: def __init__(self, name): self.name = name def speak(self): raise NotImplementedError("Subclass must implement speak()") # Child class inheriting from Animal class Dog(Animal): def speak(self): return f"{self.name} says Woof!" class Cat(Animal): def speak(self): return f"{self.name} says Meow!" # Multiple inheritance class Pet: def play(self): return f"{self.name} is playing!" class ServiceDog(Dog, Pet): def __init__(self, name, service_type): Dog.__init__(self, name) self.service_type = service_type # Using inheritance dog = Dog("Buddy") cat = Cat("Whiskers") print(dog.speak()) # Buddy says Woof! print(cat.speak()) # Whiskers says Meow! # Method Resolution Order (MRO) print(ServiceDog.__mro__)

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common parent class. It enables the same interface to work with different data types.

# Polymorphism with inheritance class Shape: def area(self): pass class Rectangle(Shape): def __init__(self, width, height): self.width = width self.height = height def area(self): return self.width * self.height class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): return 3.14159 * self.radius ** 2 # Polymorphic function def print_area(shape): print(f"Area: {shape.area()}") shapes = [Rectangle(4, 5), Circle(3)] for shape in shapes: print_area(shape) # Same method, different behavior # Duck typing - Python doesn't care about type, just behavior class Triangle: def area(self): return 0.5 * 4 * 3 print_area(Triangle()) # Works! Triangle has area() method

Abstraction

Abstraction hides complex implementation details and shows only necessary features. In Python, we use abstract classes and methods to achieve abstraction.

from abc import ABC, abstractmethod class DataProcessor(ABC): # Abstract class - cannot be instantiated directly @abstractmethod def load_data(self): pass @abstractmethod def process_data(self): pass # Concrete method def validate(self): print("Validating data...") class CSVProcessor(DataProcessor): def load_data(self): print("Loading CSV file...") def process_data(self): print("Processing CSV data...") # Cannot instantiate abstract class # processor = DataProcessor() # TypeError! processor = CSVProcessor() processor.load_data() # Loading CSV file... processor.validate() # Validating data...
Advanced

Composition

Composition is a design principle where a class is composed of other classes as members. It provides flexibility and looser coupling than inheritance.

class Engine: def start(self): return "Engine starting..." class Wheels: def rotate(self): return "Wheels rotating..." # Composition: Car HAS-A Engine and HAS-A Wheels class Car: def __init__(self, model): self.model = model self.engine = Engine() # Composition self.wheels = [Wheels() for _ in range(4)] # Composition def drive(self): print(self.engine.start()) for wheel in self.wheels: print(wheel.rotate()) return f"{self.model} is driving!" car = Car("Tesla Model 3") print(car.drive())
Advanced

Decorator Pattern

Decorators modify class behavior without changing the class itself. They're functions that wrap other functions or classes.

# Function decorator def timer(func): import time def wrapper(*args, **kwargs): start = time.time() result = func(*args, **kwargs) end = time.time() print(f"{func.__name__} took {end - start:.2f}s") return result return wrapper @timer def slow_function(): import time time.sleep(1) return "Done!" # Class decorator def logged(cls): class LoggedClass: def __init__(self, *args, **kwargs): self._instance = cls(*args, **kwargs) print(f"Created instance of {cls.__name__}") def __getattr__(self, name): print(f"Accessing {name}") return getattr(self._instance, name) return LoggedClass @logged class DataService: def fetch(self): return "Data fetched!"

Descriptor Protocol

Descriptors allow you to customize attribute access. They implement __get__, __set__, and __delete__ methods.

class RangeValidator: # Descriptor for validating numeric ranges def __init__(self, min_val=0, max_val=100): self.min_val = min_val self.max_val = max_val def __set_name__(self, owner, name): self.name = name def __get__(self, obj, objtype=None): return obj.__dict__.get(self.name, self.min_val) def __set__(self, obj, value): if not (self.min_val <= value <= self.max_val): raise ValueError(f"{self.name} must be between {self.min_val} and {self.max_val}") obj.__dict__[self.name] = value class Student: score = RangeValidator(0, 100) def __init__(self, name, score): self.name = name self.score = score # Validated! student = Student("Alice", 95) print(student.score) # 95 student.score = 150 # ValueError!
Advanced

Metaclasses

Metaclasses are the "classes of classes". They control class creation and are powerful tools for enforcing coding standards and framework development.

class Meta(type): # Metaclass - controls class creation def __new__(cls, name, bases, attrs): print(f"Creating class: {name}") # Add a unique ID to every class attrs['class_id'] = hash(name) % 1000 return super().__new__(cls, name, bases, attrs) class MyClass(metaclass=Meta): def greet(self): return "Hello!" # Output: Creating class: MyClass obj = MyClass() print(obj.class_id) # Some unique ID # Singleton pattern using metaclasses class Singleton(type): _instances = {} def __call__(cls, *args, **kwargs): if cls not in Singleton._instances: Singleton._instances[cls] = super().__call__(*args, **kwargs) return Singleton._instances[cls] class Database(metaclass=Singleton): pass db1 = Database() db2 = Database() print(db1 is db2) # True - same instance!
Advanced

Magic Methods (Dunder Methods)

Magic methods (dunder = double underscore) enable operator overloading and customize class behavior with Python's built-in operations.

class Vector: def __init__(self, x, y): self.x = x self.y = y # String representations def __str__(self): return f"Vector({self.x}, {self.y})" def __repr__(self): return f"Vector({self.x!r}, {self.y!r})" # Arithmetic operators def __add__(self, other): return Vector(self.x + other.x, self.y + other.y) def __mul__(self, scalar): return Vector(self.x * scalar, self.y * scalar) # Comparison operators def __eq__(self, other): return self.x == other.x and self.y == other.y # Length def __len__(self): return int((self.x**2 + self.y**2) ** 0.5) v1 = Vector(2, 3) v2 = Vector(4, 1) print(v1 + v2) # Vector(6, 4) print(v1 * 2) # Vector(4, 6) print(len(v1)) # 3 print(v1 == v2) # False

Data Classes

Python's @dataclass decorator automatically generates special methods like __init__, __repr__, and __eq__.

from dataclasses import dataclass, field from typing import List @dataclass class Employee: name: str salary: float department: str skills: List[str] = field(default_factory=list) # Custom method def get_bonus(self): return self.salary * 0.1 @dataclass class Team: name: str members: List[Employee] = field(default_factory=list) def total_salary(self): return sum(e.salary for e in self.members) emp1 = Employee("Alice", 75000, "Engineering", ["Python", "SQL"]) emp2 = Employee("Bob", 80000, "Data", ["PySpark"]) print(emp1) # Employee(name='Alice', ...) print(emp1 == emp2) # False print(emp1.get_bonus()) # 7500.0 team = Team("Data Team", [emp1, emp2]) print(team.total_salary()) # 155000

OOPs Summary

Classes & Objects

Blueprints and instances. Bundle data with behavior.

Encapsulation

Hide internal details. Use private/protected attributes.

Inheritance

Reuse code through parent-child relationships.

Polymorphism

Same interface, different implementations.

Abstraction

Hide complexity, show essentials via abstract classes.

Composition

Build complex objects from simpler ones.

Pro Tip: Prefer composition over inheritance for flexibility. Use inheritance for "is-a" relationships and composition for "has-a" relationships.
