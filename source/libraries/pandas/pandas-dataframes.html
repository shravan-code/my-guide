Pandas DataFrames

A focused guide to Pandas DataFrames.

What is a DataFrame? Basic

A DataFrame is a two-dimensional labeled data structure with columns of potentially different types. It is the primary workhorse of Pandas and is similar to a spreadsheet or SQL table.

Key Characteristics

  • Two-dimensional (rows × columns)
  • Heterogeneous column types
  • Labeled axes (index and columns)
  • Size mutable (rows/columns can be added)
  • Automatic data alignment

When to Use DataFrames

  • Tabular data from CSV/SQL/Excel
  • Data analysis and exploration
  • Data cleaning and transformation
  • Statistical computing
  • Machine learning preprocessing

Creating DataFrames Basic

DataFrames can be created from various sources including dictionaries, lists, files, and other data structures.

import pandas as pd import numpy as np # From a dictionary of lists df = pd.DataFrame({ 'Name': ['Alice', 'Bob', 'Charlie'], 'Age': [25, 30, 35], 'City': ['NYC', 'LA', 'Chicago'] }) # From a list of dictionaries data = [ {'Name': 'Alice', 'Age': 25}, {'Name': 'Bob', 'Age': 30} ] df = pd.DataFrame(data) # From NumPy array arr = np.random.randn(5, 3) df = pd.DataFrame(arr, columns=['A', 'B', 'C']) # From files df = pd.read_csv('data.csv') df = pd.read_excel('data.xlsx') df = pd.read_json('data.json') df = pd.read_sql('SELECT * FROM table', connection)

Selecting Data Basic

Multiple methods exist to access rows, columns, and specific cells in a DataFrame.

Column Selection

# Single column (returns Series) df['Name'] # Multiple columns (returns DataFrame) df[['Name', 'Age']] # Dot notation (only for valid identifiers) df.Name

Row Selection

# By label (.loc) df.loc[0] # First row df.loc[0:2] # Rows 0-2 # By position (.iloc) df.iloc[0] # First row df.iloc[0:2] # First 2 rows
# Cell selection df.loc[0, 'Name'] # Row 0, Name column df.iloc[0, 1] # Row 0, Column 1 # Conditional selection df[df['Age'] > 25] # Filter by condition df.query('Age > 25 and City == "NYC"') # Query string df.loc[df['Age'].idxmax()] # Row with max age # Between values df[df['Age'].between(25, 30)]

Data Manipulation Intermediate

Transform and manipulate your data with powerful DataFrame methods.

# Add new columns df['Salary'] = [50000, 60000, 70000] df['Age_in_10'] = df['Age'] + 10 df['Name_Upper'] = df['Name'].str.upper() # Drop columns or rows df.drop('City', axis=1) # Drop column df.drop(0, axis=0) # Drop row by index df.drop(columns=['A', 'B']) # Drop multiple columns # Rename columns df.rename(columns={'Name': 'Person', 'Age': 'Years'}) df.rename(columns=lambda x: x.lower()) # Sort values df.sort_values('Age', ascending=False) df.sort_values(['City', 'Age'], ascending=[True, False]) # Reset and set index df.reset_index(drop=True) df.set_index('Name') # Type conversion df['Age'] = df['Age'].astype('float32')

Grouping & Aggregation Intermediate

Group data by categories and compute aggregate statistics with groupby().

# Basic groupby grouped = df.groupby('City') # Single aggregation grouped['Age'].mean() # Mean age per city grouped['Salary'].sum() # Total salary per city grouped['Age'].agg(['mean', 'std', 'min', 'max']) # Multiple aggregations with dict grouped.agg({ 'Age': ['mean', 'max'], 'Salary': ['sum', 'mean'] }) # Transform - returns same size as original df['Age_Deviation'] = df['Age'] - grouped['Age'].transform('mean') # Filter groups df.groupby('City').filter(lambda x: x['Age'].mean() > 25) # Named aggregation (Pandas 0.25+) df.groupby('City').agg( avg_age=('Age', 'mean'), total_salary=('Salary', 'sum') )

Handling Missing Data Intermediate

Identify, analyze, and handle missing values in your DataFrame.

# Detect missing values df.isnull() # Boolean mask df.isnull().sum() # Count per column df.isnull().sum().sum() # Total missing df.info() # Summary with non-null counts # Drop missing values df.dropna() # Drop rows with any NaN df.dropna(axis=1) # Drop columns with any NaN df.dropna(thresh=2) # Keep rows with at least 2 non-null df.dropna(subset=['Name', 'Age']) # Drop if specific columns are NaN # Fill missing values df.fillna(0) # Fill with 0 df.fillna(df.mean(numeric_only=True)) # Fill with column means df.fillna(method='ffill') # Forward fill df.fillna(method='bfill') # Backward fill df.interpolate() # Linear interpolation # Replace specific values df.replace({'N/A': np.nan, '-': np.nan})

Merge & Join Intermediate

Combine multiple DataFrames using SQL-like joins.

# Inner join (default) pd.merge(df1, df2, on='id') # Different column names pd.merge(df1, df2, left_on='user_id', right_on='id') # Join types pd.merge(df1, df2, on='id', how='inner') # Only matching rows pd.merge(df1, df2, on='id', how='left') # All from left pd.merge(df1, df2, on='id', how='right') # All from right pd.merge(df1, df2, on='id', how='outer') # All from both # Index join df1.join(df2) # Join on index df1.join(df2, how='left') # Concat (stack vertically) pd.concat([df1, df2]) # Stack rows pd.concat([df1, df2], axis=1) # Stack columns

Pivot Tables Intermediate

Reshape data and create summary tables with pivot_table().

# Basic pivot table pivot = df.pivot_table( values='Sales', index='Region', columns='Product', aggfunc='sum' ) # Multiple aggregations pivot = df.pivot_table( values=['Sales', 'Profit'], index='Region', aggfunc={'Sales': 'sum', 'Profit': ['mean', 'max']} ) # Add margins for totals pivot = df.pivot_table( values='Sales', index='Region', columns='Product', aggfunc='sum', margins=True, margins_name='Total' ) # Crosstab (frequency table) pd.crosstab(df['Region'], df['Product'], normalize='index')

Apply & Map Intermediate

Apply functions to DataFrame elements, rows, or columns.

# Apply function to each column/row df.apply(lambda x: x.max() - x.min()) # Column-wise df.apply(lambda x: x.max() - x.min(), axis=1) # Row-wise # Apply with named functions def categorize_age(age): if age < 30: return 'Young' elif age < 50: return 'Middle' else: return 'Senior' df['Category'] = df['Age'].apply(categorize_age) # Map for element-wise transformation (Series only) mapping = {'NYC': 'New York', 'LA': 'Los Angeles'} df['City_Full'] = df['City'].map(mapping) # Applymap for element-wise on entire DataFrame (deprecated in 2.1+) df.map(lambda x: x.strip() if isinstance(x, str) else x) # Vectorized alternatives (faster than apply) df['Name_Upper'] = df['Name'].str.upper() # Instead of apply(str.upper) df['Age_Squared'] = df['Age'] ** 2 # Instead of apply(lambda x: x**2)

Warning: apply() is slower than vectorized operations. Use built-in methods and vectorized operations whenever possible for better performance.

Multi-Index (Hierarchical Indexing) Advanced

Multi-index allows multiple levels of indexing for complex data structures.

# Create MultiIndex DataFrame index = pd.MultiIndex.from_tuples([ ('A', 1), ('A', 2), ('B', 1), ('B', 2) ], names=['Group', 'Number']) df = pd.DataFrame({'Value': [10, 20, 30, 40]}, index=index) # Multi-level selection df.loc[('A', 1)] # Select row df.loc['A'] # Select all 'A' rows df.loc[('A':'B', 1)] # Slice outer level # Swap levels df.swaplevel() # Reset multi-index df.reset_index() # Convert index to columns df.reset_index(drop=True) # Drop index completely # Stack and unstack df.unstack() # Pivot inner level to columns df.stack() # Pivot column level to rows

Memory Optimization Advanced

Reduce DataFrame memory usage for handling larger datasets efficiently.

Check Memory Usage

# Overall memory usage df.memory_usage(deep=True) # Total memory in MB df.memory_usage(deep=True).sum() / 1024**2 # Detailed info df.info(memory_usage='deep')

Downcast Numerics

def optimize_floats(df): floats = df.select_dtypes(include=['float64']).columns df[floats] = df[floats].astype(np.float32) return df def optimize_ints(df): ints = df.select_dtypes(include=['int64']).columns df[ints] = df[ints].apply(pd.to_numeric, downcast='integer') return df
# Convert strings to category for low cardinality for col in df.select_dtypes(include=['object']): num_unique = df[col].nunique() num_total = len(df) if num_unique / num_total < 0.5: # Less than 50% unique df[col] = df[col].astype('category') # Use datetime64[ns] efficiently df['date'] = pd.to_datetime(df['date'], format='%Y-%m-%d') # Use nullable dtypes df['int_col'] = df['int_col'].astype('Int32') # Nullable int # Memory optimization function def reduce_mem_usage(df): for col in df.columns: col_type = df[col].dtype if col_type != object: c_min = df[col].min() c_max = df[col].max() if str(col_type)[:3] == 'int': if c_min > np.iinfo(np.int8).min and c_max < np.iinfo(np.int8).max: df[col] = df[col].astype(np.int8) elif str(col_type)[:5] == 'float': if c_min > np.finfo(np.float32).min and c_max < np.finfo(np.float32).max: df[col] = df[col].astype(np.float32) return df

Performance Tips Advanced

Optimize DataFrame operations for faster data processing.

Use Vectorized Operations

# ❌ Slow: Iteration for idx, row in df.iterrows(): df.loc[idx, 'result'] = row['a'] + row['b'] # ✅ Fast: Vectorized df['result'] = df['a'] + df['b']

Avoid Itertuples

# ❌ Slowest: iterrows() for row in df.iterrows(): pass # ⚠️ Better: itertuples() for row in df.itertuples(): pass # ✅ Best: Vectorized result = df['col1'] * df['col2']
# Use query() for complex filtering result = df.query('Age > 25 and City in ["NYC", "LA"]') # Use eval() for complex expressions df.eval('total = price * quantity', inplace=True) # Use categorical for groupby keys df['category'] = df['category'].astype('category') df.groupby('category').mean() # Faster with categorical # Use chunking for large files chunks = pd.read_csv('large.csv', chunksize=10000) df = pd.concat([chunk.query('value > 100') for chunk in chunks]) # Parallel processing with swifter # import swifter # df['col'] = df['col'].swifter.apply(func)

Tip: Profile your code with %timeit in Jupyter or timeit module to measure performance improvements.

Pros & Cons of DataFrames Basic

Pros

  • Rich Functionality: 200+ built-in methods for data manipulation
  • Heterogeneous Data: Different column types in one structure
  • Automatic Alignment: Data aligns by index automatically
  • I/O Integration: Read/write CSV, Excel, SQL, JSON, Parquet, etc.
  • Missing Data: Built-in NaN handling and imputation
  • Ecosystem: Integrates with matplotlib, sklearn, scipy

Cons

  • Memory Intensive: Significant overhead vs raw arrays
  • Single-threaded: No native parallelism (GIL limited)
  • Slow Iteration: iterrows() is very slow for large DataFrames
  • Copy Semantics: Chained assignment can cause issues
  • Columnar Memory: Row operations can be slower
  • Version Changes: API changes across versions can break code
