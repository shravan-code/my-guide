Databases & Data Migration

RDS, DynamoDB, and DMS for database operations and data migration.

Amazon RDS

What It Does

Managed relational database service. Supports MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora. Handles patching, backups, failover.

Why Use It

No database administration. Multi-AZ for high availability. Read replicas for scaling. Automated backups and point-in-time recovery.

Features

  • Multi-AZ: Synchronous replication for HA
  • Read Replicas: Async replication for scaling reads
  • Backups: Automated daily backups, point-in-time recovery
  • Encryption: KMS at-rest, TLS in-transit

Amazon DynamoDB

What It Does

Fully managed NoSQL database with single-digit millisecond latency at any scale. Supports both document and key-value models.

Why Use It

Serverless with on-demand capacity. Automatic scaling. Built-in caching with DAX. Global tables for multi-region.

Key Concepts

  • Partition Key: Uniqueness identifier
  • Sort Key: Hierarchical data, range queries
  • GSI: Alternate access patterns
  • DAX: In-memory caching for microsecond latency

AWS DMS

What It Does

Database migration service that moves data between databases. Supports homogeneous and heterogeneous migrations with CDC.

Why Use It

Minimal downtime migrations. Handles schema conversion. CDC for continuous replication. Supports 20+ database sources.

CDC Capabilities

  • Log-based CDC: Minimal impact on source
  • Full Load + CDC: Initial load then continuous sync
  • Validation: Verify data integrity
  • Transformation: Apply simple mappings

Use Cases

  • On-prem to cloud migration
  • Oracle to PostgreSQL/Aurora
  • Data warehouse population
  • Database consolidation
