Databases & Data Migration

RDS, DynamoDB, and DMS for database operations and data migration.

Amazon RDS

What It Does

Managed relational database service. Supports MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora. Handles patching, backups, failover.

Why Use It

No database administration. Multi-AZ for high availability. Read replicas for scaling. Automated backups and point-in-time recovery.

Features

  • check_circleMulti-AZ: Synchronous replication for HA
  • check_circleRead Replicas: Async replication for scaling reads
  • check_circleBackups: Automated daily backups
  • check_circleEncryption: KMS at-rest, TLS in-transit

Amazon DynamoDB

What It Does

Fully managed NoSQL database with single-digit millisecond latency at any scale. Supports both document and key-value models.

Why Use It

Serverless with on-demand capacity. Automatic scaling. Built-in caching with DAX. Global tables for multi-region.

Key Concepts

  • starsPartition Key: Uniqueness identifier
  • starsSort Key: Hierarchical data, range queries
  • starsGSI: Alternate access patterns
  • starsDAX: microsecond latency cache
// Table design for hot/cold access Partition Key: user_id // (for user-specific data) Sort Key: timestamp // (for time-range queries) // GSI for access patterns like "all users who bought product X" GSI PK: product_id, SK: user_id

AWS DMS

What It Does

Database migration service that moves data between databases. Supports homogeneous and heterogeneous migrations with CDC.

Why Use It

Minimal downtime migrations. Handles schema conversion. CDC for continuous replication. Supports 20+ database sources.

CDC Capabilities

  • check_circleLog-based CDC: Minimal impact on source
  • check_circleFull Load + CDC: Initial load then continuous sync
  • check_circleValidation: Verify data integrity
  • check_circleTransformation: Apply simple mappings

Use Cases

  • On-prem to cloud migration
  • Oracle to PostgreSQL/Aurora
  • Data warehouse population
  • Database consolidation
