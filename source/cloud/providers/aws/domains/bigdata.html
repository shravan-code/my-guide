EMR & Data Processing

EMR, Glue, and DataSync for large-scale data processing and migration.

Amazon EMR

What It Does

Runs Apache Spark, Hive, HBase, Flink, Presto on EC2 clusters. Full control over cluster configuration with simplified deployment.

Why Use It

For workloads too complex for Glue. ML training, large-scale ETL, streaming with Flink, and persistent HDFS requirements.

Glue vs EMR

  • Use Glue when: ETL jobs, serverless preference, occasional processing, job bookmarking
  • Use EMR when: ML training, persistent clusters, Flink streaming, HDFS needed, cost optimization with Spot

EMR Serverless

Pay per vCPU-second without managing clusters. Good for Spark workloads without cluster management overhead.

AWS Glue

What It Does

Serverless data integration service. Discovers, catalogs, and transforms data. Data Catalog (Hive metastore compatible), Crawlers, and ETL jobs.

Why Use It

No infrastructure to manage, auto-scales, pay-per-second. Handles ETL heavy lifting: schema discovery, job orchestration, error handling.

Components

  • Data Catalog: Central metadata repository
  • Crawlers: Auto-discover schema from S3, JDBC
  • ETL Jobs: PySpark/Scala Spark jobs
  • Job Bookmarks: Incremental processing
  • Workflows: Multi-job orchestration
  • Data Quality: Rule-based validation

AWS DataSync

What It Does

Online data transfer service. Moves data between on-premises storage and S3, EFS, FSx. Handles encryption, validation, scheduling.

Why Use It

10x faster than standard tools. Built-in validation. Handles large-scale migrations without impacting network.

Use Cases

  • On-prem to cloud migration
  • Data lake population
  • Cross-region replication
  • Archive to S3 Glacier
  • NAS to FSx migration
