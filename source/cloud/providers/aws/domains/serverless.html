Lambda & Event-Driven Architecture

Serverless compute and workflow orchestration with Lambda, Step Functions, and EventBridge.

AWS Lambda

What It Does

Serverless compute service that runs code in response to events. Automatically manages compute resources, scales to handle any load.

Why Use It

Pay only for compute time consumed. No servers to manage. Perfect for event-driven processing, microservices, and ETL transformations.

Limits

  • Timeout: Up to 15 minutes
  • Memory: 128MB to 10GB
  • Payload: 6MB (sync), 256KB (async)
  • Concurrent: 1000 default (soft limit)

Cold Start Times

  • Python: 100-500ms
  • Node.js: 50-300ms
  • Java: 3-10 seconds

AWS Step Functions

What It Does

Orchestration service for coordinating distributed applications. Visual workflows with built-in error handling, retries, and parallel execution.

Why Use It

No code for workflow logic. Built-in retry/compensation. Visual debugging. Integrates with 200+ AWS services.

When to Use

  • Multi-step workflows
  • Long-running processes (>15 min)
  • Human approval workflows
  • Parallel task execution
  • Distributed transaction coordination

Amazon EventBridge

What It Does

Serverless event bus that connects applications using events. Routes events from AWS services, SaaS apps, and custom sources to targets.

Why Use It

Decouples microservices, enables event-driven architectures. Native integration with 200+ AWS services and SaaS partners.

Components

  • Event Bus: Default, custom, partner buses
  • Rules: Match events and route to targets
  • Schemas: Discover and validate event structure
  • Pipes: Filter/enrich events before delivery
  • Scheduler: Cron-like scheduled triggers
