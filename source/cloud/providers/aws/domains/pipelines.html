Pipeline Examples

Real-World Data Pipelines

End-to-end pipeline architectures with AWS services. From raw data ingestion to analytics-ready datasets.

Batch ETL Pipeline

Daily processing
RDS Source DB
→
DMS CDC Extract
→
S3 Landing Zone
→
Glue ETL Job
→
S3 Curated
→
Redshift Warehouse

Daily ETL pipeline that extracts data from RDS, stages in S3, transforms with Glue, and loads to Redshift for analytics.

When to Use

Batch processing with daily or hourly intervals. Good for large datasets that don't require real-time updates. Cost-effective for predictable workloads.

Benefits

  • Cost-effective for large batch operations
  • Glue serverless - no infrastructure to manage
  • DMS handles CDC automatically
  • Redshift Spectrum can query S3 directly

Streaming Analytics Pipeline

Real-time
S3 Raw Events
→
Firehose Ingest
→
S3 Transformed
→
Athena Query
→
DynamoDB Real-time
→
QuickSight Dashboard

Real-time streaming pipeline that processes clickstream events, stores in S3 with Parquet, queries with Athena, and updates real-time dashboards.

When to Use

Sub-second latency requirements. User behavior analytics, IoT monitoring, fraud detection. Firehose provides zero-maintenance streaming with automatic scaling.

Benefits

  • Firehose zero-maintenance - no shards to manage
  • Auto-converts to Parquet/ORC for analytics
  • DynamoDB for millisecond real-time lookups
  • QuickSight for live dashboards

CDC Replication Pipeline

Change data capture
RDS Source
→
DMS CDC
→
S3 Change Data
→
Glue Apply CDC
→
Redshift Target

Capture changes from RDS and apply them to Redshift using DMS CDC. Maintains near-real-time sync with minimal latency.

When to Use

When you need to keep Redshift in sync with source databases. Supports schema evolution and filtering. Good for populating data warehouses from OLTP systems.

Benefits

  • Minimal impact on source database
  • Handles schema migrations automatically
  • Supports full load + CDC combined
  • Fault-tolerant with automatic recovery

Data Lake Architecture

Multi-source
S3 Raw Zone
→
Glue Crawler Catalog
→
Lake Formation Governance
→
Athena SQL Query
→
EMR ML/Analytics

Medallion data lake pattern with raw, curated, and analytics zones. Uses Lake Formation for fine-grained access control and Glue for ETL.

When to Use

When you need to store raw data for multiple use cases. Supports schema-on-read, ML training, ad-hoc queries, and compliance archival.

Benefits

  • Single source of truth for all data
  • Supports any data format (Parquet, ORC, JSON)
  • Fine-grained security with Lake Formation
  • Cost-effective with S3 Intelligent Tiering

EventBridge Event Bus
→
Lambda Processor
→
Step Functions Orchestration

Fully serverless event-driven pipeline using EventBridge rules, Lambda for processing, and Step Functions for complex workflows.

When to Use

Microservices architectures, workflows requiring multiple steps with error handling, and scenarios needing audit trails for each processing step.

Benefits

  • Zero server management
  • Automatic scaling
  • Built-in retry and error handling
  • Step Functions provides visual workflow debugging

SCD Type 2 Pipeline

Historical tracking
RDS Source
→
Glue SCD Logic
→
Redshift History Table

Slowly Changing Dimension Type 2 implementation that maintains full history of changes. Each change creates a new row with effective dates.

When to Use

When you need to track historical changes to dimension attributes. Common for customer data, product catalogs, and pricing history.

-- Redshift SCD Type 2 Table Structure CREATE TABLE customer_dim ( customer_key INT IDENTITY(1,1), customer_id VARCHAR(50), customer_name VARCHAR(200), email VARCHAR(200), effective_date DATE, expiry_date DATE, is_current BOOLEAN ); -- Glue: Compare incoming with current, insert new rows -- Update expiry_date and is_current for changed records

Benefits

  • Complete audit trail of all changes
  • Point-in-time analysis possible
  • Glue bookmarks enable incremental processing

ML Feature Engineering Pipeline

ML/AI ready
S3 Raw Data
→
EMR Spark Feature Eng
→
S3 Features
→
SageMaker Training
→
Endpoints Inference

End-to-end ML pipeline using EMR for feature engineering with Spark, storing features in S3, and SageMaker for training and deployment.

When to Use

Large-scale ML feature engineering that exceeds Glue's capabilities. Persistent clusters with Spot instances for cost optimization.

Benefits

  • EMR with Spot for 70% cost savings
  • SparkML for feature engineering at scale
  • SageMaker handles training and deployment
  • Feature Store integration available
