Streaming & Ingestion

Real-Time Data Streaming

Kinesis, Firehose, and MSK for real-time data ingestion and processing.

Kinesis Data Streams

What It Does

Real-time streaming data ingestion. Collects hundreds of TB per hour from logs, events, IoT sensors. Provides ordered, durable storage with millisecond latency.

Why Use It

Built for real-time at scale. Handles millions of events/second. No servers to manage, automatic scaling. Essential for real-time analytics.

Shard Capacity Math

Shards = MAX( ceil(TotalMB/s ÷ 1 MB/s), // Ingress limit ceil(TotalRecords/s ÷ 1000), // Records limit ceil(TotalMB/s × Consumers ÷ 2 MB/s) // Egress limit ) // Example: 50K events/sec, 500B each, 1 consumer Ingress: 25 MB/s = 25 shards Records: 50K/1000 = 50 shards Answer: MAX(25, 50, 25) = 50 shards

Use Cases

  • Real-time dashboards
  • IoT data ingestion
  • Clickstream analytics
  • Log processing
  • ML feature engineering

Kinesis Firehose

What It Does

Zero-maintenance streaming delivery. Captures, transforms, and loads to S3/Redshift/ES. No consumer apps needed—handles batching, compression, delivery.

Why Use It

Fully managed, auto-scaling, pay-per-GB. No shards or consumer apps. Just configure and forget.

Firehose vs Streams

  • Firehose: Zero maintenance, no replay, auto-delivery to S3/Redshift/ES
  • Streams: Custom consumers, replay capable, 1-365 day retention

Amazon MSK

What It Does

Managed Apache Kafka service. Runs Kafka clusters without managing brokers, ZooKeeper, or infrastructure. Fully compatible with Kafka ecosystem.

Why Use It

Kafka expertise not required. Handles cluster management, updates, monitoring. Integrates with MSK Connect for Kafka Connect workloads.

MSK vs Kinesis

  • MSK: Full Kafka API, offset management, topic-level control
  • Kinesis: Simpler API, enhanced fan-out, serverless option

Use Cases

  • Microservices event streaming
  • CDC from databases
  • Log aggregation
  • Real-time stream processing with Flink
