Serverless Computing

Functions, events, and managed services that scale automatically without server management.

Functions as a Service Basic

Function-as-a-Service (FaaS) lets you run code without provisioning or managing servers. Pay only for compute time.

bolt AWS Lambda

Run code for any backend service. 15 min timeout. 10GB memory limit. 1000 concurrent executions.

bolt Azure Functions

Event-driven compute. Durable Functions for stateful workflows. Multiple hosting plans.

bolt GCP Cloud Functions

HTTP and event-triggered. 2nd gen runtimes. Up to 60 min timeout. Automatic scaling.

# AWS Lambda - Python function import json def lambda_handler(event, context): return { 'statusCode': 200, 'body': json.dumps('Hello from Lambda!') } # Azure Functions - Python trigger import azure.functions as func def main(req: func.HttpRequest) -> func.HttpResponse: return func.HttpResponse("Hello from Functions!")

Event Sources Basic

Serverless functions can be triggered by various event sources from cloud services.

AWS Lambda Triggers

  • S3 (file uploads)
  • DynamoDB Streams
  • Kinesis Data Streams
  • SQS Queue messages
  • API Gateway HTTP
  • CloudWatch scheduled

Azure Functions Triggers

  • Blob Storage
  • Cosmos DB
  • Event Hubs
  • Service Bus
  • HTTP requests
  • Timer (CRON)

Common Use Cases Intermediate

Serverless excels at event-driven, bursty, and cost-sensitive workloads.

transform Data Processing

Image resizing, format conversion, ETL triggers. Scale from 0 to millions of events.

api API Backend

REST APIs, microservices, WebSocket handling. Pay per request, no idle costs.

notifications Real-time

Push notifications, chat bots, stream processing. Event-driven architecture patterns.

Limits & Considerations Intermediate

Hard Limits

  • Execution time (15-60 min)
  • Memory allocation
  • Concurrent executions
  • Payload size

Design Considerations

  • Cold start latency
  • Statelessness required
  • Vendor lock-in risk
  • Debugging complexity
