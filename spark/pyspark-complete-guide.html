1) What is Apache Spark?

Apache Spark is a distributed data processing engine designed for large-scale batch and streaming workloads. It keeps intermediate computations in memory, reducing repeated disk I/O and making iterative workloads much faster.

Input table (sales_raw)

order_idregionamount
1East100
2East150
3West80

Output table (region_total)

regiontotal_amount
East250
West80
from pyspark.sql import SparkSession
from pyspark.sql.functions import sum as Fsum

spark = SparkSession.builder.appName("spark-basics").getOrCreate()
df = spark.createDataFrame([(1,"East",100),(2,"East",150),(3,"West",80)], ["order_id","region","amount"])

region_total = df.groupBy("region").agg(Fsum("amount").alias("total_amount"))
region_total.show()

3) Complete Spark Architecture

Driver coordinates execution plans and tasks. Executors run tasks on partitions and return partial results. Cluster managers allocate resources.

Execution flow mapping

StepComponentResponsibility
1DriverBuild DAG and split into stages
2Cluster ManagerAssign executors
3ExecutorsRun tasks per partition
4DriverMerge and return action result

6) SparkContext and SparkSession

SparkContext is the low-level entry point for RDD APIs; SparkSession is the high-level unified interface for SQL/DataFrame work.

DataFrames provide schema-aware, optimized transformations. Catalyst optimizer rewrites plans for better execution.

Output (salary_band)

from pyspark.sql.functions import when, col

employees = spark.createDataFrame(
    [(1,"Eng",9000),(2,"Eng",7000),(3,"HR",5000)],
    ["emp_id","dept","salary"]
)

result = employees.withColumn(
    "salary_band",
    when(col("salary") >= 8000, "High").otherwise("Medium")
)
result.show()

8) RDDs

RDDs are low-level, immutable distributed collections with lineage-based fault recovery. They are useful for custom transformations not easily expressed in SQL/DataFrames.

lines = spark.sparkContext.parallelize([
    "spark makes big data processing fast",
    "spark supports batch and streaming"
])

word_counts = (lines
    .flatMap(lambda x: x.split())
    .map(lambda w: (w, 1))
    .reduceByKey(lambda a, b: a + b))

print(word_counts.collect())

9) Lazy Evaluation

Transformations build a logical plan and execute only when an action is called. This allows Spark to optimize the full pipeline before execution.

StepOperationExecutes immediately?
1filter()No
2select()No
3count()Yes
pipeline = df.filter("amount > 100").select("region", "amount")
# no job yet
rows = pipeline.count()  # triggers execution
print(rows)

10) Transformations

Transformations produce a new DataFrame/RDD from an existing one. Common examples: select, filter, withColumn, groupBy, join.

transformed = (df
    .filter("amount > 90")
    .withColumn("amount_tax", col("amount") * 1.18)
    .select("order_id", "region", "amount_tax"))

15) Memory Management

Spark divides executor memory for execution and storage. Misconfigured memory causes spills, GC pauses, and OOM errors.

spark.conf.set("spark.executor.memory", "8g")
spark.conf.set("spark.memory.fraction", "0.6")
spark.conf.set("spark.memory.storageFraction", "0.5")

16) Adaptive Query Execution (AQE)

AQE optimizes plans at runtime using observed statistics. It can switch join type, coalesce shuffle partitions, and split skewed partitions.

spark.conf.set("spark.sql.adaptive.enabled", "true")
spark.conf.set("spark.sql.adaptive.coalescePartitions.enabled", "true")
spark.conf.set("spark.sql.adaptive.skewJoin.enabled", "true")

17) Caching and Persistence

Cache only reused intermediate results. Choose persistence levels based on memory pressure and recomputation cost.

from pyspark import StorageLevel

heavy = df.filter("amount > 50").groupBy("region").sum("amount")
heavy.persist(StorageLevel.MEMORY_AND_DISK)
heavy.count()   # materialize cache
heavy.unpersist()

18) Broadcast Variables

Broadcast sends small reference data once to each executor, avoiding repeated transfer and reducing large shuffle joins.

from pyspark.sql.functions import broadcast

result = fact.join(broadcast(dim), on="product_id", how="left")
result.explain()

19) Structured Streaming

Structured Streaming treats stream input as an unbounded table and runs incremental micro-batch execution with SQL/DataFrame APIs.

stream_df = (spark.readStream
    .format("json")
    .schema("event_time timestamp, region string, amount double")
    .load("/data/events"))

agg = stream_df.groupBy("region").sum("amount")

query = (agg.writeStream
    .outputMode("complete")
    .format("console")
    .start())

query.awaitTermination()
